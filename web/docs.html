Methlang Language Reference

Methlang is a typed, assembly-inspired language that compiles to x86-64 assembly. This reference covers the language and toolchain.

Methlang provides explicit typing (all variables and function parameters have declared types), generic functions and structs with compile-time monomorphization, structured control flow (if, while, for, switch with break/continue), structs and enums, pointers and arrays with bounds-aware indexing, C interop via extern, a module system for importing other .meth files, and garbage collection for heap allocation via new with conservative GC runtime.

The compiler emits NASM-compatible x86-64 assembly. Linking uses the platform C runtime and any required libraries (e.g. ws2_32 for networking on Windows).

Quick Start

Get Started with Methlang in 5 Minutes

Install the compiler: Download the latest Methlang release for your platform (Windows x64 currently supported).

Write your first program: Create a file called hello.meth with the code shown below.

Compile and run: Use the build pipeline to create an executable.

Explore: See the documentation for more examples.

Lexical Structure

Low-level syntax: comments, identifiers, keywords, literals, and operators.

Comments

Line comments start with // and extend to the end of the line. Block comments are not supported. The sequence // inside a string literal is not treated as a comment; it becomes part of the string. For example, "http://example.com" produces a string containing http://example.com with no comment.

Comment examples
// Line comment: everything from // to end of line is ignored
var x: int32 = 42;  // inline comment

Identifiers

Identifiers name variables, functions, types, and other program elements. They must start with a letter or underscore, followed by any combination of letters, digits, or underscores. Case-sensitive. ASCII only; Unicode identifiers are not supported. The compiler interns identifier-like names for memory efficiency.

Valid identifiers
my_var
_private
Vector3

Keywords

Reserved words cannot be used as identifiers.

Declarations: import, extern, export, var, function, struct, enum, method. Control flow: if, else, while, for, switch, case, default, break, continue, return. Other: asm, this, new. Types: int8, int16, int32, int64, uint8, uint16, uint32, uint64, float32, float64, string.

this is only valid inside method bodies; it refers to the receiver. new is an expression keyword (e.g. var p: T* = new T); it cannot start a statement by itself. cstring is a type alias, not a keyword; it is available as a built-in name.

Numeric Literals

Decimal: 42, 0. Hexadecimal: 0x1A, 0xFF, 0Xdead. Binary: 0b1010, 0B1111. Floating-point: 3.14, 0.5, 1e-3.

A leading minus is not part of the literal. The expression -17 is parsed as the unary minus operator applied to the literal 17. So var x: int8 = -128 is valid: the literal 128 is negated to -128, which fits in int8.

Underscores in numeric literals (e.g. 1_000_000) are not supported. Use 1000000 instead.

String Literals

Strings are enclosed in double quotes. Supported escapes: \n (newline), \t (tab), \r (carriage return), \\ (backslash), \" (double quote), \0 (null byte). Unknown escape sequences are preserved literally: the backslash and the following character are both stored. For example, "\q" produces the two characters \ and q. Multiline strings are supported; a newline inside the quotes is stored as a literal newline.

String example
var msg: string = "Hello\nWorld\t\"quoted\"";

Operators and Punctuation

Assignment =. Comparison ==, !=, <, >, <=, >=. Logical &&, ||. Arithmetic +, -, *, /, %. Unary - (negation), * (dereference), & (address-of). Member access .. Arrow ->. Brackets ( ), { }, [ ]. Delimiters :, ;, ,.

Operator precedence (highest first): member access (.), multiplicative (*, /, %), additive (+, -), relational (<, <=, >, >=), equality (==, !=), logical AND (&&), logical OR (||). So a + b * c parses as a + (b * c). Use parentheses to override.

Bitwise: &, |, ^, ~, <<, >> for integer types. Unary & is address-of; binary & is bitwise AND. Context disambiguates.

Arrow ->: In function signatures it denotes the return type: function f() -> int32. In expressions it denotes pointer field access: ptr->field.

Types

Methlang is statically typed with explicit type declarations. All variables and function parameters must have a declared type.

Primitive Types

Type Size Description
int8, int16, int32, int64 1, 2, 4, 8 bytes Signed integers
uint8, uint16, uint32, uint64 1, 2, 4, 8 bytes Unsigned integers
float32, float64 4, 8 bytes IEEE 754 floating point
string 16 bytes String struct (pointer + length)
cstring 8 bytes C string pointer (null-terminated)

Type Examples

Type declarations
var age: int32 = 25;
var temperature: float64 = 98.6;

Declarations

Declarations introduce variables, functions, types, and other program elements. All declarations appear at top level or within struct bodies (for methods). A symbol must be declared before use, except for forward declarations.

Variables

Variables in Methlang are declared with the var keyword and must be explicitly typed.

Variable Declaration

Variable examples
// Variable declaration with initialization
var x: int32 = 42;
var pi: float64 = 3.14159;

// Declaration without initialization (must be assigned before use)
var result: int32;
result = 100;

// Arrays
var numbers: int32[10];
var buffer: uint8[256];
Initialization Required

Local variables must be initialized before their first use. The compiler will error if you try to read an uninitialized variable.

Functions

Functions are the primary building blocks in Methlang. They have explicit parameter types and return types.

Function Definition

Function examples
// Simple function with parameters and return value
function add(a: int32, b: int32) -> int32 {
  return a + b;
}

// Recursive function
function factorial(n: int32) -> int32 {
  if (n <= 1) {
    return 1;
  }
  return n * factorial(n - 1);
}

// Void function (no return value)
function print_message(msg: string) {
  println(msg);
}

Expressions

Expressions produce values. Operator precedence (highest first): member access (., ->), unary (-, !, *, &), multiplicative, additive, relational, equality, bitwise AND/XOR/OR, logical AND, logical OR.

Arithmetic: +, -, *, /, %. Comparison: ==, !=, <, >, <=, >=. Logical: &&, ||. Bitwise: &, |, ^, ~, <<, >>.

Unary: negation -x, logical NOT !x, dereference *p, address-of &x. Use ptr[i] for indexing; ptr + n is not supported.

Control Flow

Methlang provides structured control flow statements including conditionals, loops, and switches.

If Statements

Conditional logic
if (x > 0) {
  println("Positive");
} else if (x < 0) {
  println("Negative");
} else {
  println("Zero");
}

Loops

Loop examples
// While loop
var i: int32 = 0;
while (i < 10) {
  println_int(i);
  i = i + 1;
}

// For loop
for (var j: int32 = 0; j < 5; j = j + 1) {
  println_int(j * j);
}

Switch Statements

Switch example
switch (grade) {
  case 90:
    println("A");
    break;
  case 80:
    println("B");
    break;
  case 70:
    println("C");
    break;
  default:
    println("F");
}

Defer and Errdefer

defer schedules a statement to execute when the current scope exits, while errdefer schedules execution only when exiting with an error. Both follow LIFO (Last In, First Out) ordering.

Basic defer usage
function open_file() {
  var file: File* = fopen("data.txt", "r");
  defer fclose(file);  // Executes when function returns
  
  // ... file operations ...
}
Error handling with errdefer
function process_data() {
  var buffer: uint8* = malloc(1024);
  errdefer free(buffer);  // Only runs if function returns with error
  
  var file: File* = fopen("data.txt", "r");
  defer fclose(file);     // Always runs
  
  if (parse_error) {
    return err();  // Runs errdefer, then defer
  }
  
  return ok();     // Runs only defer
}
LIFO Ordering

Deferred statements execute in reverse order of declaration. The most recently deferred statement executes first.

Resource management pattern
function handle_client(client_socket: int32) {
  defer close_socket(client_socket);
  
  var buffer: uint8* = malloc(1024);
  errdefer free(buffer);
  
  if (read_error) {
    return err();  // Free buffer, close socket
  }
  
  return ok();     // Close socket only
}

Structs

Structs group related data together and can have methods associated with them.

Struct Definition

Struct with methods
struct Point {
  x: float64;
  y: float64;
  
  function distance(other: Point*) -> float64 {
    var dx: float64 = this.x - other->x;
    var dy: float64 = this.y - other->y;
    return dx*dx + dy*dy; // Simplified distance
  }
  
  function move(dx: float64, dy: float64) {
    this.x = this.x + dx;
    this.y = this.y + dy;
  }
}

function main() -> int32 {
  var p1: Point = {0.0, 0.0};
  var p2: Point = {3.0, 4.0};
  
  var dist: float64 = p1.distance(&p2);
  p1.move(1.0, 2.0);
  
  return 0;
}

Enums

Enums define a named type with a set of variants, each having an integer value.

Enum Definition

Enum examples
enum Direction {
  North,        // 0
  East = 2,     // 2
  South,        // 3 (previous + 1)
  West = -5     // -5
}

enum Status {
  Pending = 100,
  InProgress = 200,
  Complete = 300,
  Failed = 400
}

function main() -> int32 {
  var dir: Direction = North;
  var status: Status = InProgress;
  
  if (dir == East) {
    println("Facing East");
  }
  
  switch (status) {
    case Complete:
      println("Task completed!");
      break;
    case Failed:
      println("Task failed!");
      break;
  }
  
  return 0;
}

Enum Properties

Enum Characteristics
  • Underlying Type: Enums use int64 as the underlying representation (8 bytes, aligned to 8)
  • Auto-increment: Variants without explicit values continue from previous variant (0 if first)
  • Direct Usage: Variant names are in scope after enum definition (use North, not Direction.North)
  • Comparisons: Enums can be compared with integers and used in switch cases

Type Conversion

Integer to enum conversion
enum Priority {
  Low = 1,
  Medium = 5,
  High = 10
}

function set_priority(level: int32) -> Priority {
  // Implicit narrowing conversion
  var priority: Priority = level;
  
  if (priority >= High) {
    println("High priority set");
  }
  
  return priority;
}

Modules

Methlang provides a module system for organizing code and managing dependencies. Modules allow you to split large programs into manageable, reusable components.

Import System

Use the import statement to include modules in your program.

Import examples
// Import standard library modules
import "std/io";
import "std/mem";
import "std/math";

// Import custom modules (relative paths)
import "utils/string_helpers";
import "datastructures/list";
import "algorithms/sort";

// Import with absolute path (less common)
import "/usr/local/lib/custom";

Module Resolution

Import Search Order

The compiler resolves imports in this order:

  1. Relative to current file: "utils/helpers" looks for utils/helpers.meth
  2. Include directories: Paths added with -I flag
  3. Standard library: std/ prefix searches in the bundled stdlib (shipped with the compiler)

Creating Modules

Creating a custom module
// File: utils/string_helpers.meth

// Export functions for use by other modules
export function string_length(s: string) -> int32 {
  return s.length;
}

export function string_equals(a: string, b: string) -> int32 {
  if (a.length != b.length) {
    return 0; // false
  }
  
  var i: int32 = 0;
  while (i < a.length) {
    if (a.chars[i] != b.chars[i]) {
      return 0; // false
    }
    i = i + 1;
  }
  
  return 1; // true
}

// Export types for use by other modules
export struct StringBuilder {
  buffer: uint8[1024];
  length: int32;
  
  function append(text: string) {
    // Implementation...
  }
}

Using Custom Modules

Using custom modules
// File: main.meth

import "utils/string_helpers";

function main() -> int32 {
  var message: string = "Hello, Methlang!";
  var length: int32 = string_length(message);
  
  println("Message length: ");
  println_int(length);
  
  var builder: StringBuilder;
  builder.length = 0;
  builder.append(message);
  
  return 0;
}

Module Best Practices

Module Guidelines
  • Single Responsibility: Each module should have a clear, focused purpose
  • Explicit Exports: Use export keyword for functions/types meant to be used externally
  • Avoid Circular Dependencies: Module A shouldn't import module B if B also imports A
  • Consistent Naming: Use descriptive names and consider prefixing with module category
  • Documentation: Comment module purpose and exported function behavior

Imports

Methlang provides module imports (import) and embedded file imports (import_str). Both use the same path resolution rules.

Module Import

import "std/io" loads another Methlang source file. Paths use forward or backslashes. If no extension, .meth is appended. Imported declarations are flattened into the global scope. Only exported declarations are visible when a module uses export.

Embedded File Import

import_str "index.html" reads a file at compile time and embeds its contents as a string. Use for HTML, config files, or any text. No automatic extension. Result has .chars and .length.

Path Resolution

1. Absolute paths used as-is. 2. Paths starting with std/ resolve under the stdlib root (bundled with the compiler by default; override with --stdlib for a custom stdlib). 3. Relative paths are relative to the importing file's directory. 4. -I adds import search directories.

Garbage Collector

Methlang provides an optional conservative mark-and-sweep garbage collector. Programs that use new must link the GC runtime (gc.c). Programs using only stack allocation and C malloc do not need it.

Use new for managed heap allocation (struct instances, dynamic data). Use C malloc for unmanaged memory (I/O buffers, C interop).

Linking the GC

Build with GC
methlang main.meth -o main.s
nasm -f win64 main.s -o main.o
gcc -c src/runtime/gc.c -o gc.o -Isrc
gcc main.o gc.o -o main

Manual Memory (malloc)

Using C malloc/free
import "std/mem";

function main() -> int32 {
  // Allocate memory
  var buffer: cstring = malloc(1024);
  
  if (buffer != 0) {
    // Use the memory
    memset(buffer, 0, 1024);
    
    // Free when done
    free(buffer);
  }
  
  return 0;
}

Garbage Collection

GC Optional

When you link the GC runtime, you can use new T for automatic memory management. The GC uses conservative mark-and-sweep collection.

Using garbage collection
struct Node {
  value: int32;
  next: Node*;
}

function main() -> int32 {
  // Allocate with garbage collection
  var node: Node* = new Node;
  node->value = 42;
  node->next = new Node;
  
  // No need to free - GC will clean up
  return 0;
}

C Interoperability

Methlang provides seamless C interoperability through extern declarations.

Declaring External Functions

C function declarations
// Declare C functions
extern function strlen(s: cstring) -> int32 = "strlen";
extern function strcpy(dest: cstring, src: cstring) -> cstring = "strcpy";
extern function malloc(size: int32) -> cstring = "malloc";

function main() -> int32 {
  var message: uint8[256];
  strcpy(&message[0], "Hello from C!");
  
  var len: int32 = strlen(&message[0]);
  println_int(len);
  
  return 0;
}

Compilation

Usage: methlang [options] <input.meth>. The input file is the main source. Imports are resolved relative to it. Default output is output.s.

Standard library

You do not need a stdlib/ folder in your project. The compiler uses the bundled stdlib that ships with it by default. Use --stdlib <dir> only when you intentionally want a custom stdlib.

Options

-o <file> output assembly. -i <file> input file. -I <dir> add import search directory (repeatable). --stdlib <dir> use a custom stdlib (default: bundled stdlib). --prelude auto-import std/prelude. -d/--debug debug mode and embedded runtime crash traceback support. -g/--debug-symbols generate debug symbols. -l/--line-mapping source line mapping. -s/--stack-trace embed runtime crash traceback support without the rest of debug mode. -O/--optimize enable optimizations. -r/--release enables -O, strips comments, removes unreachable functions, disables runtime null/bounds checks.

Compilation Pipeline

1. Lexing 2. Parsing 3. Import resolution 4. Monomorphization 5. Type checking 6. IR lowering 7. Optimization (optional) 8. Code generation

Build Pipeline

Compile: methlang main.meth -o main.s

Assemble: nasm -f win64 main.s -o main.o (or -f elf64 on Linux)

Link: gcc -nostartfiles main.o gc.o -o main -lkernel32. Use -nostartfiles so Methlang's entry point is used. If using new, compile and link src/runtime/gc.c.

Runtime Crash Tracebacks

Compile with -s or -d to embed runtime crash traceback support. On Windows, generated programs print native exception codes such as 0xC0000005 with Meth function and source frames when metadata is available.

Example crash output
Unhandled runtime exception 0xC0000005 (access violation)
Exception address: 0x00007FF7DFD71046
write access violation at 0x0000000000000001
Stack trace:
  #0 leaf_crash at app.meth:2:3 (0x00007FF7DFD71046)
  #1 intermediate at app.meth:6:3 (0x00007FF7DFD71080)
  #2 main at app.meth:10:3 (0x00007FF7DFD710A0)

Known Limitations

Current limitations of the Methlang language and compiler.

  • No block comments. Only line comments (//).
  • No top-level constant expressions. Use functions that return constant values.
  • Generic type parameters are unconstrained. No trait bounds.
  • switch case values must be compile-time constant integers. No range cases.
  • No compound assignment (+=, -=). Use x = x + 1.
  • No labeled break or continue.
  • No pointer arithmetic with ptr + n. Use indexing ptr[i].
  • Runtime null/bounds checks disabled in --release.
  • Deferred calls capture by reference, not by value.
  • No conditional imports.
  • std/net and web server are Windows-only (Winsock2).

Common Issues

"Undefined reference to `gc_alloc'"

Cause: Using new expressions without linking the GC runtime.

Solution: Compile and link src/runtime/gc.c with your program:

Linking with GC
gcc -c src/runtime/gc.c -o gc.o -Isrc
gcc main.o gc.o -o main
"Cannot open output file: Permission denied"

Cause: The output executable is currently running or locked.

Solution: Stop any running instances of the program before rebuilding.

"Use before initialization" error

Cause: Reading from a local variable before assigning it a value.

Solution: Always initialize variables before use:

Variable initialization
// Wrong - using uninitialized variable
var x: int32;
return x; // Error!

// Correct - initialize before use
var x: int32 = 0;
return x; // OK

Runtime Issues

Program crashes on startup

Cause: Missing GC initialization when using new expressions.

Solution: Ensure the entry point calls gc_init() (handled automatically when linking with GC runtime).

Need Better Crash Diagnostics

Compile with -s or -d to embed runtime crash traceback support. On Windows this reports native exception codes and prints Meth stack frames for null dereferences, bounds traps, and access violations when frame metadata is available.

Performance slower than expected

Possible causes:

  • Not using optimization flag (-O)
  • Frequent GC collections (consider raising threshold)
  • Excessive memory allocations in hot paths

Solutions: Use optimizations, profile memory usage, consider stack allocation for performance-critical code.
